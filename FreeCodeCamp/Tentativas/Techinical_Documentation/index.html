Introdução

JavaScript é uma linguagem de script orientada a objetos e multiplataforma. É uma linguagem pequena e leve. Dentro de um ambiente host (por exemplo, um navegador da web), JavaScript pode ser conectado aos objetos de seu ambiente para fornecer controle programático sobre eles.

JavaScript contém uma biblioteca padrão de objetos, como Array, Date e Math, e um conjunto central de elementos da linguagem, como operadores, estruturas de controle e declarações. O JavaScript básico pode ser estendido para uma variedade de propósitos ao suplementá-lo com objetos adicionais; por exemplo:

  • JavaScript do lado do cliente estende a linguagem principal fornecendo objetos para controlar um navegador e seu Modelo de Objeto de Documento (DOM). Por exemplo, extensões do lado do cliente permitem que um aplicativo posicione elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e navegação na página.
  • JavaScript do lado do servidor estende a linguagem principal fornecendo objetos relevantes para a execução de JavaScript em um servidor. Por exemplo, extensões do lado do servidor permitem que um aplicativo se comunique com um banco de dados, forneça continuidade de informações de uma invocação para outra do aplicativo ou execute manipulações de arquivos em um servidor.

JavaScript e Java

JavaScript e Java são semelhantes em alguns aspectos, mas fundamentalmente diferentes em outros. A linguagem JavaScript se assemelha ao Java, mas não possui a tipagem estática e a verificação rigorosa de tipos do Java. O JavaScript segue a maioria da sintaxe de expressões do Java, convenções de nomenclatura e estruturas básicas de controle de fluxo, o que foi a razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o sistema de classes do Java, que é construído em tempo de compilação por meio de declarações, o JavaScript suporta um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores numéricos, booleanos e strings. JavaScript tem um modelo de objetos baseado em protótipos, em vez do modelo mais comum baseado em classes. O modelo baseado em protótipos fornece herança dinâmica; ou seja, o que é herdado pode variar para objetos individuais. JavaScript também suporta funções sem requisitos declarativos especiais. As funções podem ser propriedades de objetos, executando-se como métodos de tipagem dinâmica.

JavaScript é uma linguagem muito mais flexível em comparação com Java. Você não precisa declarar todas as variáveis, classes e métodos. Não precisa se preocupar se os métodos são públicos, privados ou protegidos, e não precisa implementar interfaces. Variáveis, parâmetros e tipos de retorno de função não são explicitamente tipados.

Olá mundo

Para começar a escrever JavaScript, abra o Scratchpad e escreva seu primeiro código "Hello world" em JavaScript:

function greetMe(yourName) { alert("Hello " + yourName); } greetMe("World");

Selecione o código no bloco e pressione Ctrl+R para vê-lo sendo executado no navegador!

Você usa variáveis como nomes simbólicos para valores em seu aplicativo. Os nomes das variáveis, chamados de identificadores, seguem certas regras.

Um identificador JavaScript deve começar com uma letra, sublinhado (_) ou cifrão ($); os caracteres subsequentes também podem ser dígitos (0-9). Como JavaScript diferencia maiúsculas de minúsculas, as letras incluem os caracteres "A" a "Z" (maiúsculas) e "a" a "z" (minúsculas).

Você pode usar letras ISO 8859-1 ou Unicode, como å e ü, em identificadores. Também pode usar sequências de escape Unicode como caracteres em identificadores. Alguns exemplos de nomes legais são Number_hits, temp99 e _name.

Declarar variáveis

Você pode declarar uma variável de três maneiras:

Com a palavra-chave var. Por exemplo:

var x = 42;

Essa sintaxe pode ser usada para declarar variáveis locais e globais.

Simplesmente atribuindo um valor. Por exemplo:

x = 42;

Isso sempre declara uma variável global. Gera um aviso estrito do JavaScript. Você não deve usar essa variante.

Com a palavra-chave let. Por exemplo:

let y = 13;

Essa sintaxe pode ser usada para declarar uma variável local com escopo de bloco. Veja Escopo de variáveis abaixo.

Escopo de variáveis

Quando você declara uma variável fora de qualquer função, ela é chamada de variável global, porque está disponível para qualquer outro código no documento atual. Quando você declara uma variável dentro de uma função, ela é chamada de variável local, porque está disponível apenas dentro dessa função.

JavaScript antes do ECMAScript 2015 não tem escopo de bloco; em vez disso, uma variável declarada dentro de um bloco é local para a função (ou escopo global) em que o bloco está contido. Por exemplo, o código a seguir registrará 5, porque o escopo de x é a função (ou contexto global) onde x foi declarado, não o bloco, que, neste caso, é uma instrução if.

if (true) { var x = 5; } console.log(x); // 5

Esse comportamento muda quando se usa a declaração let, introduzida no ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Variáveis globais

As variáveis globais são, na verdade, propriedades do objeto global. Em páginas da web, o objeto global é window, então você pode definir e acessar variáveis globais usando a sintaxe window.variable.

Consequentemente, você pode acessar variáveis globais declaradas em uma janela ou quadro a partir de outra janela ou quadro especificando o nome da janela ou quadro. Por exemplo, se uma variável chamada phoneNumber for declarada em um documento, você pode referenciar essa variável de um iframe como parent.phoneNumber.

Constantes

Você pode criar uma constante nomeada e somente leitura com a palavra-chave const. A sintaxe de um identificador constante é a mesma que a de um identificador de variável: deve começar com uma letra, sublinhado ou cifrão e pode conter caracteres alfabéticos, numéricos ou sublinhados.

const PI = 3.14;

Uma constante não pode mudar de valor por atribuição nem ser redeclarada enquanto o script estiver em execução. Ela deve ser inicializada com um valor.

As regras de escopo para constantes são as mesmas que para variáveis com escopo de bloco declaradas com let. Se a palavra-chave const for omitida, o identificador será assumido como uma variável.

Você não pode declarar uma constante com o mesmo nome de uma função ou variável no mesmo escopo. Por exemplo:

// ISSO CAUSARÁ UM ERRO function f() {}; const f = 5; // ISSO TAMBÉM CAUSARÁ UM ERRO function f() { const g = 5; var g; // declarações }

No entanto, os atributos de objetos não são protegidos, então a seguinte declaração será executada sem problemas:

const MEU_OBJETO = {"chave": "valor"}; MEU_OBJETO.chave = "outroValor";

Tipos de dados

O padrão mais recente do ECMAScript define sete tipos de dados:

  • Seis tipos de dados primitivos:
    • Boolean. true e false.
    • null. Uma palavra-chave especial que denota um valor nulo. Como o JavaScript diferencia maiúsculas de minúsculas, null não é o mesmo que Null, NULL ou qualquer outra variação.
    • undefined. Uma propriedade de nível superior cujo valor é undefined.
    • Number. 42 ou 3.14159.
    • String. "Olá".
    • Symbol (novo no ECMAScript 2015). Um tipo de dado cujas instâncias são únicas e imutáveis.
  • E o tipo Object.

Embora esses tipos de dados sejam relativamente poucos, eles permitem que você execute funções úteis em suas aplicações. Objetos e funções são os outros elementos fundamentais da linguagem. Você pode pensar nos objetos como contêineres nomeados para valores e nas funções como procedimentos que sua aplicação pode executar.

Introdução if...else

Use a instrução if para executar um bloco de código se uma condição lógica for verdadeira. Use a cláusula opcional else para executar um bloco de código se a condição for falsa. Uma instrução if se parece com o seguinte:

if (condição) { declaração_1; } else { declaração_2; }

if (condição_1) { declaração_1; } else if (condição_2) { declaração_2; } else if (condição_n) { declaração_n; } else { declaração_final; }

Quando há múltiplas condições, apenas a primeira que for avaliada como true será executada. Para executar múltiplas declarações, agrupe-as dentro de um bloco { ... }.

Em geral, é uma boa prática sempre usar blocos {} ao aninhar instruções if:

if (condição) { declaração_1_é_executada_se_condição_for_verdadeira; declaração_2_é_executada_se_condição_for_verdadeira; } else { declaração_3_é_executada_se_condição_for_falsa; declaração_4_é_executada_se_condição_for_falsa; }

É recomendável evitar atribuições simples dentro de expressões condicionais, pois podem ser confundidas com comparações de igualdade. Por exemplo, não use:

if (x = y) { /* declarações aqui */ }

Se for necessário fazer uma atribuição em uma expressão condicional, uma prática comum é colocar parênteses extras:

if ((x = y)) { /* declarações aqui */ }

Introdução while

Uma instrução while executa suas declarações enquanto uma condição especificada for avaliada como true. A estrutura de uma instrução while é a seguinte:

while (condição) declaração;

Se a condição se tornar false, as declarações dentro do loop param de ser executadas e o controle passa para a instrução seguinte ao loop.

A condição é testada antes de a declaração no loop ser executada. Se a condição for true, a declaração é executada e a condição é testada novamente. Se a condição for false, a execução para e o controle passa para a instrução seguinte a while.

Para executar múltiplas declarações, use um bloco { ... } para agrupá-las.

Exemplo:

O seguinte loop while itera enquanto n for menor que 3:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

A cada iteração, o loop incrementa n e adiciona esse valor a x. Portanto, x e n assumem os seguintes valores:

  • Após a primeira execução: n = 1 e x = 1.
  • Após a segunda execução: n = 2 e x = 3.
  • Após a terceira execução: n = 3 e x = 6.

Após concluir a terceira execução, a condição n < 3 não é mais verdadeira, então o loop termina.

Declaração de funções

Uma definição de função (também chamada de declaração de função ou instrução de função) consiste na palavra-chave function, seguida de:

  • O nome da função.
  • Uma lista de argumentos da função, entre parênteses e separados por vírgulas.
  • As instruções JavaScript que definem a função, entre chaves { }.

Por exemplo, o código a seguir define uma função simples chamada

function quadrado(numero) { return numero * numero; }

A função quadrado recebe um argumento chamado numero. Ela contém uma única instrução que retorna o argumento multiplicado por ele mesmo.

return numero * numero;

Parâmetros primitivos (como números) são passados para funções por valor; ou seja, o valor é passado para a função, mas se a função alterar o valor do parâmetro, essa alteração não será refletida globalmente nem na função chamadora.
